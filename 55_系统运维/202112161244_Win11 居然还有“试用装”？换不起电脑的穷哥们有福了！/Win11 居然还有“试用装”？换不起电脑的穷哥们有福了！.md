### 用 `Docker` 来玩

官方良心，为我们穷小白提供了极致的服务体验，他们将这个好玩的东西做成了 `Docker` 镜像。

这下可好了，有 `Docker` 的地方都能玩了。

比如你在局域网内，又不方便直接连接外网，那么恭喜你，你很走运，完全可以在局域网里做到在线体验 `Win11` 了。

具体怎么做，很简单！



首先，你要有 `Docker` ，这个就不啰嗦了，请小伙伴们自行搞定吧。



其次，拉取官方镜像。

```
docker pull blueedge/win11react:latest
```



我这儿拉取的是17个小时前的最新镜像。

图22



最后，运行容器，将服务放在后台，映射开放端口为 `3000` ，当然你也可以改成其他的。

```
docker run -d --restart unless-stopped --name win11react -p 3000:3000 blueedge/win11react:latest
```

图23



好了来吧，地址栏里输入 `Docker` 所在服务器的 IP 并加端口号。

```
http://server_ip:3000
```

回车看一下，嘿，完美运行“ `Win11` ”，和前面的官方 `win11.blueedge.me` 一样一样的哈！

图24



接下来就和前面在线版的一样玩耍就行了。

什么，你说你没有 `Docker` ？

难办了吧？

不难办，哥们还有一招！



### 单机体验 `Win11`

前面我们演示的都是 `Web` 版 `Win11` ，实际上大神们手里还藏着一个单机版，`Windows` 、 `Linux` 和 `MacOS` 都能玩。

在这儿我们就以 `Windows` 平台为例，到以下链接下载相应的 `Win11React` 安装包。

```
https://github.com/blueedgetechno/win11React/releases
```

找到那个 `msi` 包将它下载下来。

图25



下载好了没？

需要注意的是，这个安装包不是下载后直接双击安装就可以的，如果直接安装它会报错。

我测试后总结了经验，在安装这个 `msi` 包之前，我们需要先安装 `winget` 。

`winget` 这玩意我也不常用，有点像 `Linux` 下的包管理软件。

安装 `winget` 的方法很多，较为方便快捷的方式就是到应用商店中获取。

按下 `Win` + `R` 键弹出运行框，在其中输入以下命令直接定位 `winget` 。

```
ms-windows-store://pdp/?productid=9nblggh4nns1
```
